## 6. Visual Identity

### 6.1 Color Palette

Personal brand colors should feel: **technical, premium, slightly dark-mode, game-adjacent without being "gamer".**

| Role | Color | Hex | Usage |
|---|---|---|---|
| **Primary** | Deep Navy | `#0D1B2A` | Backgrounds, headers, dark sections |
| **Accent** | Electric Teal | `#00C9B1` | CTAs, highlights, links, key stats |
| **Secondary** | Warm White | `#F5F5F0` | Body text on dark, light backgrounds |
| **Neutral** | Slate Gray | `#6B7C93` | Secondary text, metadata, dates |
| **Highlight** | Amber Gold | `#F5A623` | Awards, standout achievements, badges |
| **Dark bg** | Near Black | `#070E14` | Full-bleed dark sections |

**Rationale:** The navy/teal palette signals technical depth (common in developer tooling and infra brands) while the amber accent calls out achievements without being neon. Avoids the red/green "gamer" cliché while still feeling at home in the games industry.

**Prohibited:** No gradients on name/title text. No neon green or RGB keyboard rainbow aesthetics. No pure black (#000000) backgrounds.

---

### 6.2 Typography

| Role | Font | Weight | Size | Usage |
|---|---|---|---|---|
| **Display / Name** | Inter | 800 ExtraBold | 48–64px | Name on CV header, hero sections |
| **Section headers** | Inter | 700 Bold | 18–24px | Section titles |
| **Job titles** | Inter | 600 SemiBold | 14–16px | Role names, company names |
| **Body text** | Inter | 400 Regular | 12–14px | Descriptions, achievements |
| **Code / Tech terms** | JetBrains Mono | 400 Regular | 12–13px | Tech stacks, tool lists |
| **Metadata** | Inter | 300 Light | 11–12px | Dates, locations, links |

**System fallback stack:** `Inter, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`

**Rationale:** Inter is the de facto standard for technical/developer profiles — clean, readable, modern without being trendy. JetBrains Mono for code references signals developer authenticity.

---

### 6.3 Layout Principles

- **Information density: medium-high.** Recruiters scan; CTOs read. The layout must survive both modes.
- **Left-aligned, columnar structure** for CV. Two columns max (main content + sidebar for skills/tools).
- **Section hierarchy is mandatory.** Primary sections (Experience, Projects) always above Skills lists.
- **Whitespace is not wasted.** Minimum 8px between list items, 24px between sections.
- **Skill lists must be grouped**, never flat alphabetical dumps.
- **Dates always right-aligned** or in a consistent metadata style — never free-floating.

---

### 6.4 Profile Photo Guidelines

- **Style:** Professional but approachable. Plain or minimal background preferred (dark or neutral).
- **Framing:** Head and shoulders. Direct eye contact with camera.
- **Expression:** Confident, not stiff. No forced smile.
- **No:** Gaming chairs in background, heavy filters, overly casual setting.
- **Dimensions:** Use square crop (1:1) for LinkedIn and profile contexts; allow for 4:3 or 3:4 crop on CV.

## 7. Voice & Tone Matrix

### 7.1 Core Voice Attributes

| Attribute | In practice |
|---|---|
| **Ownership language** | "I led / I built / I designed / I operated" — never "contributed to" or "helped with" for primary responsibilities |
| **Precision over volume** | Name the system, the tool, the number. "Matchmaking and seasonal ranking in Nakama" beats "multiplayer features" |
| **Outcomes over activity** | "Led to an active player base with live tournaments" beats "Developed backend systems" |
| **Active voice always** | "I deployed infrastructure on AWS using Terraform" not "Infrastructure was deployed" |
| **No filler adjectives** | Remove: passionate, hardworking, dedicated, dynamic, results-driven. Replace with evidence. |

### 7.2 Tone by Channel

| Channel | Tone | Example |
|---|---|---|
| **CV** | Precise, dense, factual. No personality required — let outcomes speak. | "Built matchmaking, ranking ladder, and seasonal systems. Led team of 7." |
| **LinkedIn** | Direct and professional, slightly warmer. First person. No corporate speak. | "I build multiplayer games and the infrastructure to run them. Here's what that looks like in practice." |
| **Email / outreach** | Direct, short, specific. One ask per email. | "I saw you're hiring for a Unity backend engineer. I've been building and operating a live service card game for 3 years. Worth 20 minutes?" |
| **Portfolio / website** | Confident and narrative. Tell the project story. Show the decisions. | "Dawnshard needed a backend that could handle ranked seasons, matchmaking, and a live economy — I designed and built the full Nakama architecture." |
| **Teaching / talks** | Accessible, structured. Break complexity down. | First establish the concept, then the application, then the tradeoff. |

### 7.3 Forbidden Phrases

Never use these in Federico's brand materials:

| ❌ Forbidden | ✅ Replace with |
|---|---|
| "Passionate game developer" | State what you've built |
| "Strong communication skills" | "Coordinated a 7-person team across development and production" |
| "Results-oriented professional" | Give the actual result |
| "Familiar with" | Either you've used it in production (say so) or cut it |
| "Helped develop" | "Developed" — own it |
| "Various tools and technologies" | Name the specific ones |
| "Fast learner" | Show a project where you picked something up and shipped it |
